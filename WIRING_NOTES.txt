========================================================
 Phase 1 wiring — changes to EXISTING files
========================================================
These are the only edits needed in files already in the
repo to connect the two new Phase 1 files.

--------------------------------------------------------
FILE: ApplicationSettings.cs
--------------------------------------------------------

1. Remove the old Properties.Settings reference.

   FIND (line ~37):
       this.Settings = Builder.Presentation.Properties.Settings.Default;

   REPLACE WITH:
       this.Settings = AppSettingsStore.Load();

2. Change the Settings property type declaration.

   FIND (line ~42):
       internal Builder.Presentation.Properties.Settings Settings { get; }

   REPLACE WITH:
       internal AppSettingsStore Settings { get; private set; }

   (It needs a setter now because Reload() reassigns it.)

3. Update the Reload() method.

   FIND:
       public void Reload() => this.Settings.Reload();

   REPLACE WITH:
       public void Reload()
       {
           this.Settings.Reload();
           // Notify the UI that all bound settings may have changed
           this.OnPropertyChanged(string.Empty);
       }

   (OnPropertyChanged("") is the standard way to signal
    all properties changed at once in INotifyPropertyChanged.)

4. Remove the using for Properties.Settings (if present):

   REMOVE (if it appears at the top of the file):
       using Builder.Presentation.Properties;

No other changes are needed in ApplicationSettings.cs —
every this.Settings.XYZ call will now hit AppSettingsStore.

--------------------------------------------------------
FILE: ApplicationManager.cs
--------------------------------------------------------

1. Remove SetSecurityProtocol() — obsolete in .NET 8.

   DELETE the entire method:
       public void SetSecurityProtocol()
       {
           ServicePointManager.SecurityProtocol =
               SecurityProtocolType.Tls11 | SecurityProtocolType.Tls12;
       }

2. Remove the call to it in App.xaml.cs.

   In App.xaml.cs, FIND:
       current.SetSecurityProtocol();

   DELETE that line.

3. Remove the now-unused using at the top of ApplicationManager.cs:

   REMOVE:
       using System.Net;

   (ServicePointManager and SecurityProtocolType live there.)

4. Update UpgradeConfigurationCheck() to call the new migration helper.

   FIND:
       public void UpgradeConfigurationCheck()
       {
           Builder.Presentation.Properties.Settings settings =
               Builder.Presentation.Properties.Settings.Default;
           if (!settings.ConfigurationUpgradeRequired)
               return;
           try
           {
               settings.Upgrade();
               settings.ConfigurationUpgradeRequired = false;
               settings.Save();
           }
           catch (Exception ex)
           {
               AnalyticsErrorHelper.Exception(ex, ...);
           }
       }

   REPLACE WITH:
       public void UpgradeConfigurationCheck()
       {
           AppSettingsStore store = this.Settings.Settings;
           if (!store.ConfigurationUpgradeRequired)
               return;
           AppSettingsStore.MigrateFromLegacyConfig(store);
       }

5. Replace the Windows-Explorer Process.Start calls in
   ValidateConfiguration() with the cross-platform helper.

   ADD this private method anywhere in the class:

       private static void OpenDirectory(string? path)
       {
           if (string.IsNullOrWhiteSpace(path)) return;
           if (OperatingSystem.IsWindows())
               System.Diagnostics.Process.Start("explorer.exe", path);
           else if (OperatingSystem.IsMacOS())
               System.Diagnostics.Process.Start("open", path);
           else
               System.Diagnostics.Process.Start("xdg-open", path);
       }

   Then replace every occurrence of:
       Process.Start(fileInfo.DirectoryName)

   With:
       OpenDirectory(fileInfo.DirectoryName)
